## Description

A person’s health data is stored in various sources, both internal to a single facility (radiology department, emergency department, laboratory, etc.) and external (pharmacy, home health care, family doctor’s office, connected devices, etc.). In addition to the fragmentation of data across numerous sources that are poorly, inadequately, or not at all interconnected, there are also issues of interoperability and tracking changes in the data. To assist clinicians, researchers, and administrators in dealing with the increasing number of stakeholders and the variety of available data, it is essential to produce a unified view of the data that is semantically annotated at different levels of granularity.

## Objectives

- Develop equivalence rules between ontology components and relational components;
- Develop equivalence rules between ontology components and graph components;
- Develop optimization rules for relational and graph structures;
- Develop rules for mapping data sources to the knowledge model;
- Develop a user interface to assist in mapping data sources to the knowledge model;
- Develop query construction algorithms from an ontology;
- Develop a user interface to assist users in defining queries;
