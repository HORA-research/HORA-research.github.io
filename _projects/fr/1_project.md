## Description

Les données relatives à la santé d’une personne existent dans plusieurs sources internes à un même établissement (service de radiologie, service d’urgence, laboratoire, etc.) ou externes (pharmacie, soins à domicile, cabinet du médecin de famille, objets connectés, etc.). À la dispersion des données dans de nombreuses sources peu, mal ou pas interconnectées s’ajoutent le problème d’interopérabilité et celui du suivi de l’évolution des données. Pour assister les personnes cliniciennes-chercheuses de même que les gestionnaires face à l’augmentation du nombre d’intervenants et de la variété de données disponibles, il est indispensable de produire une vue unifiée des données annotée sémantiquement à différents niveaux de granularité.

## Objectifs

-	Élaborer les règles d’équivalences entre les composants d’une ontologie et les composants relationnels ;
Élaborer les règles d’équivalences entre les composants d’une ontologie et les composants de graphes ;
-	Élaborer les règles d’optimisation de la structure relationnelle et de graphes ;
- Élaborer les règles de mise en correspondances des sources de données au modèle de connaissances ;
- Élaborer une interface personne-machine pour assister à la mise en correspondances des sources de données au modèle de connaissances ;
- Élaborer des algorithmes de construction de requêtes à partir d’une ontologie ;
- Élaborer une interface personne-machine pour assister un utilisateur dans la définition de requêtes ;

## Logiciel
